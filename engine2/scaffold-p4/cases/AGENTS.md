# Cases AGENTS.md

Cases are clinical learning artefacts.

Agents may:
- format locked cases
- migrate one locked case at a time
- validate schema
- prepare status objects
- prepare Source Basis Maps from approved sources

Agents must not:
- generate missing locked case content from memory
- alter clinical meaning
- use raw feedback as clinical source
- treat old drafts as final
- migrate Case 2 or Case 3 without lock packet and canonical bundle
- generate Case 4 without gate pass or exact bypass phrase

Case folders must contain:
- case.md
- case.json
- status.json
- source-basis-map.yaml
- qa.json
- voice.json
